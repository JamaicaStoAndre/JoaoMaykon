Manual de Orientação para Log:
Adicionando um novo log:

Determine o tipo/nível do log (erro, aviso, informação, etc.).
Adicione um novo transporte no logger principal, especificando o nome do arquivo e o nível.
Se necessário, defina um formato para filtrar mensagens específicas.
Utilizando funções de log existentes:

Utilize logIntentNotFound(message) para registrar erros relacionados a intenções não encontradas.
Use logSqlError(message, additionalInfo) para registrar erros SQL.
Use logWarning(message, additionalInfo) para registrar avisos.
Utilize logSuccess(message, additionalInfo) para registrar mensagens de sucesso.
Extensão:

Ao estender o sistema de log, lembre-se de manter a estrutura de diretórios organizada e de nomear claramente os arquivos de log.
Verificação:

Revise regularmente os arquivos de log, especialmente o logGeral.log, para se manter atualizado sobre o status e possíveis problemas do sistema.
Backup:

Realize backups regulares dos arquivos de log para garantir que os dados não sejam perdidos.
